# ORS（手术室事项安排系统）开发计划

## 目标与约束
- **目标**：快速上线一个以**手机端为主**的 ORS 网站，兼顾桌面端，支持“每日交班（填报/查看/导出打印）”“培训教程（二维码直达）”“账号登录与权限”。
- **约束**：单人开发，优先成熟组件与模板；可在**宝塔面板**上快速部署与维护。
- **组织架构**：支持多科室扩展，采用三层角色：**系统 root 管理员 → 科室管理员 → 科室成员**。

## 推荐技术栈（宝塔友好 / 快速落地）
- **后端**：Laravel 11（PHP 8.2+）
- **管理后台**：Filament 3（用于管理员侧：员工/课程/交班数据管理、导出入口等）
- **鉴权/账号**：
  - Laravel Breeze（登录/重置密码等基础功能）
  - spatie/laravel-permission（角色权限：管理员/成员）
- **数据库**：MySQL 8（宝塔原生）
- **导出**：
  - maatwebsite/excel（Excel）
  - barryvdh/laravel-dompdf（PDF 打印版式）
- **二维码**：simplesoftwareio/simple-qrcode（交班入口码、课程二维码）
- **前端/UI**：
  - 成员端：Blade + Tailwind CSS（移动端“类 App”体验）
  - 管理端：Filament 自带响应式 UI（省时且美观）

## 功能范围（MVP 优先级）
### P0（必须先做：能交班、能导出、能管人）
1. **账号/科室/权限（root-科室管理员-成员）**
   - root 管理员：创建/禁用科室、指派科室管理员、跨科室查看与导出
   - 科室管理员：管理本科室成员账号（创建/重置/禁用/移除）、管理本科室交班与课程
   - 科室成员：仅查看/填报本科室数据
2. **每日交班**
   - 成员扫码/链接进入后：登录 → 进入“今日交班”
   - **交班填报二维码固定不变**：二维码始终指向 `/m/handover/today`，系统按服务器当天日期自动进入当日交班
   - **卡片式交班浏览**：一台手术信息一张卡，支持左右滑动/按钮切换，显示进度（x / N）
   - 成员可**新增/填报**当日手术信息（字段按纸质表格）
   - 管理员可按日期汇总，导出 **Excel / PDF**（PDF 用于直接打印归档，版式接近纸表）
3. **交班入口二维码**
   - “今日交班”固定入口（二维码不变，自动定位当天日期）
   - 可选：管理员选择某日生成二维码（用于补录历史）

### P1（第二阶段：培训教程闭环）
1. **培训课程管理**
   - 管理员创建课程：标题、正文（文字+图片）、视频（Bilibili 等平台嵌入）
2. **课程二维码**
   - 每个课程生成唯一访问地址 + 二维码
   - 成员扫码登录后直接进入课程详情页学习

### P2（体验增强/运维完善，按需）
- 表单草稿、缺项提示、字段模板复用
- 交班场次“锁定/归档”
- 导出模板细节优化（分页、横向 A4、字体、线条）
- 操作日志/审计（谁新增/谁修改）
- 数据备份策略（宝塔计划任务）

## 核心业务设计（建议定稿）
### 交班数据组织方式
- 每天一个“**交班日期/场次**”（通常按日期即可）
- 当日包含多条“**手术条目**”（每条=一张交班卡片）

### 交班卡片展示（手机端主流程）
- 默认进入“今日交班”卡片流：
  - 全屏卡片、左右滑动切换
  - 进度提示（3/18）
  - 信息分区折叠：基础信息 / 交接要点 / 评估单 / 器械准备 / 费用与其它

### 员工账号生命周期（管理员侧）
- 创建账号（手动录入或自动生成初始密码）
- 重置密码（展示一次/复制）
- **优先禁用**（保留历史记录归属）；必要时再软删除

## 里程碑计划（单人、以“能上线”为标准）
> 下面按“最短路径”推进，优先打通交班闭环。

### 阶段 A：基础工程与登录（1–2 天）
- Laravel 初始化、数据库连接
- Breeze 登录（含记住登录）
- 角色权限（管理员/成员）
- Filament 管理后台接入
- 管理员：员工 CRUD、重置密码、禁用

**交付物**
- 管理员可创建/禁用员工账号
- 成员可登录进入系统首页（移动端布局）

### 阶段 B：每日交班 MVP（3–6 天）
- 数据模型：交班日期（或直接 date 字段）+ 手术条目
- 成员端：
  - 今日交班卡片流（查看全部条目）
  - 新增/编辑自己填报的手术条目
- 管理员端：
  - 按日期筛选、汇总列表
  - 导出 Excel
  - 打印 PDF（版式接近纸质表格）
- 交班入口二维码：固定“今日交班”链接码

**交付物**
- 扫码 → 登录 → 卡片式交班浏览 → 填报 → 管理员导出并打印

### 阶段 C：培训教程 + 二维码（2–4 天）
- 课程 CRUD（图文 + 视频嵌入）
- 课程二维码生成与访问控制
- 移动端课程详情页（阅读/播放体验）

**交付物**
- 管理员发布课程并生成二维码；成员扫码学习

### 阶段 D：上线到宝塔（0.5–1 天）
- 宝塔：Nginx + PHP 8.2 + MySQL
- 站点目录指向 `public`
- 配置伪静态、`storage` 链接、环境变量
- 备份策略（数据库与 `storage`）

## 页面与路由（建议）
- `/login` 登录
- `/m` 移动端首页（今日交班、培训入口）
- `/m/handover/today` 今日交班卡片流
- `/m/handover/{date}` 指定日期交班（可选）
- `/m/handover/items/create` 新增手术条目
- `/m/courses` 课程列表
- `/m/courses/{id}` 课程详情（二维码指向此）
- `/admin` 管理后台（Filament）

## 数据表（建议草案）
- `departments`：科室
- `department_members`：用户-科室关系（科室内角色 admin/member）
- `users`：账号、姓名、状态（active/disabled）等；root 作为全局角色存在
- `handover_sessions`：交班日期（按科室）
- `handover_items`：交班手术条目（归属 session）
- `courses`：课程（按科室）

## 部署建议（宝塔）
- PHP：8.2
- 数据库：MySQL 8
- 站点：Nginx，根目录 `.../public`
- 计划任务：每日数据库备份
